技术路线说明

回答方为 中数元宇。当前 SDK 的 AprilTag 检测采用 自研 DeepApriltag 深度学习方案，不同于传统二值化阈值 + 轮廓检测路线；检测与解码由神经网络完成 Tag 定位、角点亚像素回归和 ID 解码，再通过 PnP 与非线性重投影精修输出位姿。该路线面向游戏人物和击剑交互中的运动模糊、极端角度、快速尺度变化、强弱光切换、反光和较大比例遮挡等工况。

1. 检测网络定位画面中 Tag 候选区域或热力图。
2. 角点精修网络完成亚像素角点回归，增强抗模糊和抗反光能力。
3. ID 解码完成编码识别与纠错，输出稳定 Tag ID。
4. 位姿优化通过 PnP + 非线性重投影精修输出位姿。
工况传统 AprilTag 风险当前 DeepTag 路线
反光 / 弱暗光容易漏检通过数据增强和学习式检测提升鲁棒性
10%~50% 遮挡多数场景失效角点回归可做补全，保留可检能力
运动模糊 / 弯曲贴附角点抖动大亚像素角点精修提升稳定性
端侧实时性CPU 即可但鲁棒性有限需要 NNAPI / NPU-RK3588 等端侧加速，已纳入工程化

该路线带来数据集、训练、模型压缩部署和位姿精修等额外工作量，但这是游戏人物运动模糊、极端角度和快速尺度变化场景下提升识别稳定性的必要投入。

Q3 Tag 选型 实测决策

Tag 选型围绕尺寸、材质、数量和组合方式展开，目标是在剑体 / 护具等击剑场景部件上保持多角度可检。中数元宇将以选型报告 + 实测方式给出结论，并提供备用码型与图纸；最终选型以真机实测为准。

维度初步建议
Tag 尺寸参考 20~40mm；近距小尺寸、远距大尺寸，可多尺寸混用。最终通过光照、运动、尺度变化、距离等条件交叉验证后选取表现最好的尺寸。
Tag 材质参考标定板材质方案，需抗反光、漫反射、可贴合、耐磨耐汗。
Tag 数量剑体多枚、护具多枚冗余分布，保证任意视角至少 1~2 个 Tag 可见；具体数量通过实测确定。
组合方式多 Tag 按刚体分组、ID 段分配，区分剑体和护具，便于多 Tag 融合位姿。

光速侧需提供剑体与护具的可粘贴区域尺寸和方式，并确认是否允许多尺寸 / 冗余布点。深度学习模型将针对最终选定码型与材质做适配训练。

需求文档与验收场景 验收口径

阶段 1 验收参考标准包括：1 Tag 正对稳定叠加、遮挡 25% / 50% 可检、多 Tag 同时检测、暗光 / 正常光 / 强光、0.3m 到 3m 距离范围，以及 30 分钟无泄漏不降帧。新版 README 明确指出 75% 遮挡过高，当前不建议作为标准验收项，因为可能显著降低识别率。

验收专项游戏 / 场景需要在需求文档中明确。目前场景倾向为击剑，但仍需最终确定；中数元宇将据此准备 Demo 与同步态调试 API。
