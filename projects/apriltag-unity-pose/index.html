最新技术答复整理

技术路线说明

当前 SDK 的 AprilTag 检测采用 DeepTag 深度学习检测路线，整体路线对标 DeepArUco / DeepArUco++：检测网络首先定位 Tag 候选区域或热力图，角点精修网络进一步完成亚像素角点回归，随后执行 ID 解码与纠错，最后通过 PnP 与非线性重投影精修输出位姿。该路线用于覆盖传统二值化阈值与轮廓检测方案在反光、弱暗光、运动模糊、弯曲贴附和较大比例遮挡下的失效风险。

工况传统 AprilTag 风险当前 DeepTag 路线
反光 / 弱暗光容易漏检通过数据增强和学习式检测提升鲁棒性
10%~50% 遮挡多数场景失效角点回归可做补全，保留可检能力
运动模糊 / 弯曲贴附角点抖动大亚像素角点精修提升稳定性
端侧实时性CPU 即可但鲁棒性有限需要 NNAPI/NPU 等端侧加速，已纳入工程化工作

Q1 真机依赖

阶段 1 的联调、相机标定和性能测试必须在创维真机上完成。算法、插件和 Demo 主体可以先在 PC / 安卓开发机与离线仿真环境中推进，但真机 bring-up、端侧推理帧率和 30 分钟稳定性测试不能由仿真替代。建议至少提供 2 台真机，分别用于常驻开发联调和测试留观。

Q2 图像与标定接口

需创维侧明确取流 API 或回调方式、分辨率、帧率、像素格式和是否支持零拷贝；同时需提供相机内参、畸变参数、外参 / 坐标系、帧时间戳、曝光/增益读取能力和 IMU 可用性。若只提供标定途径而非标定结果，则阶段 1 需要安排实测标定工作。

Q3 Tag 选型

Tag 尺寸建议以 22~28mm 为初始实验区间，最终根据真机实测确定；贴附材质需满足抗反光、曲面贴合、耐磨和耐汗要求；数量和组合方式采用多 Tag 冗余分布，并按刚体分组和 ID 段进行管理。
