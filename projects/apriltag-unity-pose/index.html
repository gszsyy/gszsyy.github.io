进度摘要

阶段 1 目标

  • 完成多面体结构与 Tag 贴附方案验证。
  • 从单帧灰度图像中检测可见面 Tag，并融合为刚体位姿。
  • 将 6DoF 位姿输出给 Unity，使虚拟物体跟随实体结构运动。
  • 路线从传统 AprilTag 推进到 DeepTag，提升低光、运动模糊、反光和遮挡下的鲁棒性。

已完成事项 阶段验证

  • 已在创维 1600 上完成初步验证，可以进行基础驱动。
  • 在未接入创维底层的条件下，通过分时复用方式拿到约 30fps 灰度图像。
  • 当前图像帧无法严格对应 IMU 数据，但大致精度与稳定性已经完成测试。
  • 偶发卡顿仍存在，主要受取流方式、帧同步和底层接口限制影响。
  • 阶段 1 待确认问题已在参考 README 中完成答复，真机、标定、取流和 Tag 选型被列为关键依赖。

1. 结构与 Tag 方案确认 8 方向可见面、Tag 尺寸、贴附材质与 ID 规划。任意 45 度方向至少一个 Tag 可见。结构与参考材料已具备
2. DeepTag 检测检测网络、角点精修、ID 解码、低光/运动模糊数据增强。弱光、反光、遮挡与模糊下稳定输出角点。路线已确定，需继续端侧优化
3. 位姿融合多 Tag 刚体融合、PnP、非线性重投影精修、坐标系转换。输出稳定 xyz 平移与旋转。基础验证完成
4. 创维 1600 联调接入底层取流、相机标定、帧时间戳和 IMU 同步。降低卡顿，提升时序稳定性。当前未接底层，存在风险
5. Unity Demo 验收虚拟物体跟随实体结构运动，输出日志和 QC 指标。30 分钟稳定性、帧率、漏检率、位姿误差。下一步

风险与下一步

当前风险

  • 创维 1600 尚未接入底层取流，分时复用导致偶发卡顿。
  • 灰度帧无法严格对应 IMU 数据，运动补偿和时序融合受限。
  • 低光和运动模糊已转向 DeepTag，但仍需要真机场景数据覆盖。

下一步

  • 拿到创维底层取流、相机内参、畸变参数和时间戳接口。
  • 对 STL/STP 结构进行 Tag 坐标系标定，固化每个面的外参。
  • 完成 DeepTag 端侧推理压缩与实时性测试。
  • 建立 Unity 驱动日志，输出延迟、丢帧、重投影误差和位姿抖动指标。
